# EvE VS Code拡張 テスト手順 v0.2.0

## 1. ストリーミング表示

### テスト手順
1. F5で拡張をデバッグ起動
2. サイドバーの「EvE」アイコンをクリック
3. 「こんにちは」と入力して送信
4. **期待結果**: 「🤔 考え中」が表示され、応答が1文字ずつストリーミングで表示される
5. 完了後、Markdownレンダリングされること

### 確認ポイント
- [ ] 思考中インジケーターが表示される
- [ ] ストリーミングでリアルタイム表示
- [ ] 完了後にMarkdownが正しくレンダリング（コードブロック、リスト等）

## 2. ファイルコンテキスト自動添付

### テスト手順
1. Pythonファイルを開く（例: test.py）
2. チャットで「このファイルのバグを見つけて」と入力
3. **期待結果**: AIがファイル内容を認識して回答
4. 📎ボタンをクリックして「ファイル添付: OFF」に変更
5. 再度質問 → **期待結果**: ファイル内容なしの回答

### 確認ポイント
- [ ] 自動添付ON時、ファイル内容が送信される
- [ ] 自動添付OFF時、ユーザー入力のみ送信
- [ ] 📎ボタンでトグルできる

## 3. 日本語セットアップガイド

### テスト手順
1. コマンドパレット → `EvE: セットアップウィザード`
2. **期待結果**: チェック画面が表示される
3. 各項目のOK/NGを確認
4. 「再チェック」ボタンをクリック → 再度確認
5. 「閉じる」ボタンで閉じる

### 確認ポイント
- [ ] Ollama起動確認が正しく判定
- [ ] eve-cliパス確認が正しく判定
- [ ] モデル一覧が表示される
- [ ] すべてOK時「準備完了！」表示
- [ ] NG時は修正手順が日本語で表示

## 4. Markdown レンダリング

### テスト手順
1. チャットで「Pythonのフィボナッチ関数を書いて」と入力
2. **期待結果**: コードブロックにシンタックスハイライト付きで表示
3. リストや見出しを含む回答も正しくレンダリングされること

### 確認ポイント
- [ ] コードブロックにハイライト適用
- [ ] 見出し（#）、リスト（-）、太字（**）が正しく表示
- [ ] ユーザーメッセージはプレーンテキスト（Markdownなし）

## 5. モデル切替

### テスト手順
1. 「切替」ボタンをクリック
2. モデル一覧から `gemma4:31b-cloud` を選択
3. **期待結果**: ヘッダーのモデル名が更新される
4. メッセージ送信 → 新しいモデルで応答

### 確認ポイント
- [ ] QuickPickでモデル選択可能
- [ ] 選択後、ヘッダー表示が更新
- [ ] 選択モデルで実際に応答が変わる
